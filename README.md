# hy-home.k8s: 지능형 홈랩의 새로운 기준

`hy-home.k8s`는 단순한 서버 구축을 넘어, 사람과 AI가 문서와 코드로 대화하며 함께 만들어가는 스마트 홈랩 프레임워크입니다. 모든 과정은 **Spec-Driven Development (SDD)**를 기반으로 하며, 01-11 단계로 나뉜 촘촘한 문서 체계를 통해 설계부터 운영까지의 모든 맥락을 완벽하게 추적합니다.

## 🚀 프로젝트의 심장, 문서 체계

우리 프로젝트의 모든 고민은 `docs/` 폴더 안에 숫자로 체계화되어 있습니다. AI는 이 구조를 통해 프로젝트의 맥락을 읽고, 우리는 이 구조를 통해 프로젝트의 현재를 한눈에 파악합니다.

- **00. 거버넌스 (Agent)**: AI가 어떻게 행동해야 하는지 정의한 가이드라인 ([상세 보기](docs/00.agent-governance/README.md))
- **01. 제품 요구사항 (PRD)**: "왜 이 기능을 만드는가?"에 대한 근본적인 질문과 대답
- **02~03. 설계와 결정 (ARD/ADR)**: 아키텍처의 원형과 기술적 선택의 이유들
- **04. 상세 명세 (Spec)**: 구현을 위한 단 하나의 진실(SSoT)
- **05~06. 계획과 작업 (Plan/Task)**: 마일스톤 설계와 실제 작업 진행 상황
- **07~11. 운영 및 회고 (Ops)**: 운영 정책, 런북, 그리고 사고로부터 배우는 교훈들

## 📐 우리가 소통하는 방식 (언어 원칙)

명확하고 효율적인 협업을 위해 언어의 역할을 나눴습니다.

1. **AI 에이전트용 (Internal English)**: 에이전트가 더 정확하게 생각하고 도구를 사용할 수 있도록, 내부 지침과 기술 명세 상세는 **영어**로 작성합니다.
2. **사람용 (External Korean)**: 우리가 직관적으로 이해하고 소통할 수 있도록, 모든 README와 개요, 그리고 사용자와의 대화는 **한국어**로 진행합니다.
3. **답변 원칙**: 어떤 언어로 요청하든, 최종 답변은 항상 **한글**로 제공합니다.
