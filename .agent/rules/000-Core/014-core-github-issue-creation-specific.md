# GitHub Issue Creation Standards

## 1. Process

When creating issues via AI or automation:

1. **Analyze**: Understand the feature request or bug report.
2. **Research**: Check existing issues and `CONTRIBUTING.md`.
3. **Plan**: Outline the issue structure (Title, Description, Acceptance Criteria).
4. **Draft**: Generate the issue content in Markdown.

## 2. Structure

- **Title**: Clear and descriptive (e.g., "Add User Authentication via OAuth").
- **Description**: Detailed explanation of the problem or feature.
- **Acceptance Criteria**: Checklist of what "done" looks like.
- **Context**: Screenshots, error logs, or relevant links.
- **Labels**: Apply `bug`, `enhancement`, `documentation`, etc.

## 3. Tone

- **Professional**: Maintain a professional and constructive tone.
- **Actionable**: Ensure the issue is actionable for a developer.
