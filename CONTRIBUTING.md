# Contributing to [Project Name]

Thank you for your interest in contributing!

## Development Workflow

1. **Create a Plan**: All features start with a plan.
    - Run `npm run setup <spec-path>` to initialize your environment for a specific feature.
2. **Follow the Pillars**:
    - **Standards**: See `agent_settings/rules/0100-Standards`.
    - **Testing**: Ensure 100% coverage for new logic.

## Pull Requests

- **Branching**: Use `feature/` or `fix/` prefixes.
- **Commits**: Use Conventional Commits (e.g., `feat: login page`).
- **Review**: All code must pass the [Code Review Standard](file:///agent_settings/rules/0000-Agents/0012-code-review-standard.md).
